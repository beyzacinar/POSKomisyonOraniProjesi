
# Machine Learning-Assisted Debit Ratio-Based POS Commission Rate Determination System

## About the Project

This project was developed to determine customer-specific and data-driven POS commission rates for merchants applying for POS services through digital channels.

Since new customers do not have historical POS transaction data, banks may face difficulties in providing customer-specific commission pricing. Therefore, new customers are often offered standard commission rates or the maximum legally permitted rates. This may result in non-competitive pricing, potential customer loss, and reduced customer satisfaction.

The developed system estimates the expected debit card transaction ratio of a merchant by using merchant characteristics, sector information, POS type, customer segment, and transaction-related variables. The estimated debit ratio is then evaluated together with the bank’s transaction cost advantage to support the determination of an appropriate POS commission rate.

## Project Objective

Debit card transactions generally generate lower costs for banks than credit card transactions. Therefore, as the debit card transaction ratio of a merchant increases, the bank gains a transaction cost advantage.

The main objectives of the project are:

- To estimate the debit card transaction ratio of new POS customers,
- To calculate the cost advantage generated by debit transactions,
- To produce customer-specific and competitive POS commission rate recommendations,
- To support customer acquisition while maintaining bank profitability,
- To transform manual pricing processes into a data-driven decision support system.

## Applied Methods

The following machine learning algorithms were evaluated within the scope of the project:

- Linear Regression
- Hist Gradient Boosting Regressor
- XGBoost Regressor
- Random Forest Regressor

According to the model comparison results, the Random Forest Regressor achieved the best overall performance.

## Data Preprocessing

The following preprocessing steps are applied in the Python implementation:

1. Removing identifier and period columns that are not required for model training,
2. Filling missing numerical values with the median,
3. Filling missing categorical values with the most frequent value,
4. Encoding categorical variables using `LabelEncoder`,
5. Splitting the dataset into 80% training data and 20% test data,
6. Training the Random Forest Regressor model,
7. Evaluating model performance using regression metrics,
8. Generating a feature importance chart.

## POS Commission Rate Determination Approach

The debit ratio estimated by the machine learning model is evaluated together with the cost difference between debit and credit card transactions.

The commission rate determination approach is expressed using the following function:

```text
Recommended Commission Rate =
3.56% − α × Estimated Debit Ratio × 0.23%
```

Where:

- `3.56%` represents the maximum commission rate considered for credit card transactions,
- `0.23%` represents the additional gross margin advantage generated by debit transactions,
- `α` represents the bank’s pricing and profitability policy parameter,
- `Estimated Debit Ratio` represents the output of the machine learning model.

The policy parameter may be adjusted according to different pricing strategies:

| α Value | Strategy |
|---:|---|
| 1.0 | The entire cost advantage is transferred to the customer |
| 0.5 | The cost advantage is shared between the bank and the customer |
| 0.0 | The maximum commission rate is preserved |
